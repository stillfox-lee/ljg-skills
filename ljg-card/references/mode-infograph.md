# 模具：信息图（-i）

## 视觉哲学

实验室精密手册感 + 波普实验风格。拒绝平庸手账，追求数据可视化的极致美感。

核心原则：**内容的结构决定视觉形式**。没有预设模块——先看内容是什么形状，再设计最贴合的视觉。

## 步骤 1：读取模板

Read `~/.claude/skills/ljg-card/assets/infograph_template.html`

模板提供 shell（header、footer、色彩变量、网格背景）和一套工具类。内容区域是空白画布，所有视觉由你根据内容定制。

**模板工具类速查**：

| 类名 | 用途 |
|------|------|
| `.label` | 大写等宽微标签（10px），可加 `--green` / `--pink` 变体 |
| `.body` | 正文排版（16px/1.8，#4a4a4a，max-width 62ch） |
| `.big-num` | 英雄数字（52px 等宽），`.big-num--lg` 为 72px |
| `mark` | 荧光笔高亮 |
| `.tag` | 内联标签，粉色左边框；`.tag--dark` 为暗底变体 |
| `.serif` | 衬线字体切换（Georgia/Noto Serif SC） |
| `.full-bleed` | 全宽条带——突破 canvas padding，用于深色/绿色横条 |
| `.strip-dark` | 深色底 + 白字 |
| `.strip-green` | 绿色底 |
| `.gap-sm/md/lg/xl` | 间距块（16/24/40/56px） |
| `.divider` / `.divider--heavy` | 分隔线 |

使用 `var(--sp-xs)` 到 `var(--sp-2xl)` 作为间距 token，禁止随意数字。

## 步骤 2：内容分析

- 如为 URL/文件：获取内容，提炼核心
- 如为文本：直接分析
- 目标：**识别信息的内在结构**，而非提取填充物

### 2.1 提取元信息

- **标题**：从内容提炼（≤ 15 字）
- **副标题**：一句话概括核心（≤ 30 字）
- **来源**：作者或出处（用户提供则用，否则 `李继刚`）

### 2.2 结构识别

**先问：这段信息的形状是什么？**

| 结构类型 | 识别信号 | 适合的视觉形式 |
|----------|---------|---------------|
| 循环/飞轮 | A→B→C→A，正反馈 | 环形图、螺旋、闭合箭头链 |
| 对比/对照 | 旧vs新、A vs B | 左右分屏、before/after、双轨 |
| 层级/金字塔 | 底层支撑上层 | 三角、倒金字塔、同心圆 |
| 流程/序列 | 先后顺序、步骤 | 时间轴、管道流、泳道图 |
| 分布/比例 | 数值对比、占比 | 条形图、面积块、刻度尺 |
| 矩阵/分类 | 两个维度交叉 | 2×2 象限、网格、坐标系 |
| 辐射/因果 | 一个核心多个衍生 | 星形放射、树枝、hub-spoke |
| 堆叠/积累 | 层层叠加 | 地层图、堆叠块、千层饼 |

**多数内容是复合结构**：主结构 + 辅助结构。例如"两个飞轮的对比"= 对比（主）+ 循环（子）。

识别后输出：

```
主结构：[类型] — [一句话描述]
子结构：[类型] — [一句话描述]（如有）
视觉方案：[一句话描述你打算怎么画]
```

## 步骤 3：视觉设计

### 3.1 色彩系统（固定）

不需选色。模板内置固定色盘：

| 角色 | 色值 | 用途 |
|------|------|------|
| 蓝图底 | `#F2F2F2` | 背景 + 网格纹理 |
| 鼠尾草绿 | `#B8D8BE` | 主功能区块、稳定区域、正面 |
| 荧光粉 | `#E91E63` | 关键警告、最重要单点、胜出项 |
| 柠檬黄 | `#FFF200` | 荧光笔高亮关键词（半透明） |
| 炭褐 | `#2D2926` | 线条、文字、网格、坐标 |

**色彩使用策略**——60/30/10 法则：
- **60%** 蓝图底 + 白色（中性承托）
- **30%** 绿色 + 炭褐（结构与内容）
- **10%** 荧光粉 + 柠檬黄（仅用于最关键的一两个点）

粉色是子弹，不是油漆——用在最值得看的那一处。

### 3.2 排版戏剧性

信息图的排版必须有**张力**，不是均匀分布：

**字号跳跃**：层级之间至少 3× 差异。标题 64px 配正文 16px 是对的；标题 28px 配正文 16px 是平的。

**字重对比**：900 配 400，不是 600 配 500。轻重交替制造呼吸。

**混排**：等宽（`.label`、数据）+ 无衬线（正文）+ 衬线（`.serif`，引文/金句）。三种字体各司其职，禁止全篇一种。

**数字突出**：所有关键数值用 `.big-num`（等宽 52px），与正文形成视觉锚点。没有大数字的信息图是失败的——找到值得放大的那个数。

### 3.3 布局设计

根据识别出的结构，设计整体布局。画布宽 1080px，高度自适应。

**设计原则**：
- 主结构占据视觉重心（面积 ≥ 60%）
- 辅助信息围绕主结构排布，不喧宾夺主
- 用 CSS Grid 实现**非对称布局**：`2fr 1fr`、`1fr 380px`、`3fr 2fr`——禁止等分
- 允许全宽条带（`.full-bleed`）——深色/绿色横条打断节奏，制造高潮
- 循环结构用 CSS 画圆弧/箭头，不用图片

**节奏感**：一张好的信息图像一首曲子，有松有紧：

```
轻   → 词源/背景（小字、留白、时间轴）
重！ → 核心公式/发现（全宽深色条带、大字）
中   → 展开解释（正文密度、左右分栏）
重！ → 关键对比/数据（非对称色块、大数字）
收   → 金句/结论（衬线、留白、呼吸）
```

不是每张图都走这个序列，但必须有**至少一个高潮**（全宽条带或深色块）和**至少一处留白**。

### 3.4 内容密度要求

- 关键数值用 `.big-num` 突出
- 关键词用 `<mark>` 实现荧光笔高亮
- 使用视觉符号增加精密感（Σ、Δ、∞、→、×）
- 坐标编号可选：若内容适合实验室编目感则用，不适合则不强加

**禁止**：
- 不要生成「关键词/标签」类填充区块（信息密度极低）
- 不要为了凑数量而添加模块——3 个高质量区块好过 7 个稀薄区块

### 3.5 反 AI 俗套清单

生成 CSS 前，逐项排查以下致命模式：

| 俗套 | 为什么是俗套 | 替代方案 |
|------|------------|---------|
| 三等分等宽卡片 | AI 生成的头号标志 | 非对称：`2fr 1fr`，或一大二小叠放 |
| 居中 Hero 标题 | 毫无构图意图 | 左对齐、分屏、偏置 |
| 圆角卡片 + 灰色阴影 | 模板套件默认值 | 用 `border-top` / 纯留白分组 / 染色阴影 |
| 每个区块背景都是白色 | 单调、无节奏 | 交替使用白色、绿色、深色底 |
| 所有文字一个字号 | 没有层级 | 至少 3 级字号跳跃（3× 差异） |
| `border-radius: 8px` 全场 | 无意图的默认值 | 要么大圆角 `2.5rem`、要么直角 `0`、有意识地选择 |
| padding/margin 随机数字 | 间距混乱 | 只用 `var(--sp-*)` token |

### 3.6 全宽条带用法

全宽条带（`.full-bleed`）是信息图的**高潮工具**，用来承载最重要的一句话、核心公式或关键数据。

```html
<div class="full-bleed strip-dark" style="padding: var(--sp-xl) var(--canvas-px);">
  <div class="label label--green">Σ 核心意象</div>
  <!-- 大字内容 -->
</div>
```

规则：
- 一张信息图最多 **2 个** 全宽条带（否则高潮通胀）
- 深色条带内用绿色 `.label` 和白色文字
- 绿色条带内用炭褐文字

## 步骤 4：生成 HTML

将自定义 CSS 写入 `{{CUSTOM_CSS}}`，将内容 HTML 写入 `{{CONTENT_HTML}}`。

替换模板变量：

| 变量 | 内容 |
|------|------|
| `{{TITLE}}` | 主标题 |
| `{{SUBTITLE}}` | 副标题 |
| `{{REF_CODE}}` | 动态参考编码，格式：`REF—{类别缩写} / {内容关键词大写}`。例：`REF—AI / AGENT ARCHITECTURE`、`REF—BIO / PROTEIN FOLDING`。从内容中提取领域和核心主题生成，不要用固定值 |
| `{{CUSTOM_CSS}}` | 根据内容定制的所有 CSS |
| `{{CONTENT_HTML}}` | 根据结构设计的所有内容 HTML |
| `{{SOURCE}}` | 来源/署名 |
| `{{ARXIV_LINE}}` | 内容来自 arxiv 论文时：`<span class="arxiv">arxiv: XXXX.XXXXX</span>`；否则空字符串 |

写入：`/tmp/ljg_cast_infograph_{name}.html`

## 步骤 5：质量自检

生成 HTML 后逐项确认：

**结构与内容**：
- [ ] 视觉形式是否贴合内容的内在结构？（飞轮用环形，对比用分屏……）
- [ ] 主结构是否占据视觉重心（面积 ≥ 60%）？
- [ ] 关键位置是否有具体数值或 `.big-num`？
- [ ] 信息密度是否足够？（没有填充块、没有空洞区域）

**排版层级**：
- [ ] 至少 3 级字号且层级间 ≥ 3× 差异？
- [ ] 字重对比明显（900 vs 400，不是 600 vs 500）？
- [ ] 混排了等宽 + 无衬线 + 衬线（至少两种）？
- [ ] 正文颜色是 #4a4a4a 或类似深灰，非纯黑？

**布局与节奏**：
- [ ] 无三等分等宽卡片？
- [ ] 无居中 Hero？
- [ ] 有至少一个全宽条带或深色块（高潮点）？
- [ ] 有至少一处留白（呼吸点）？
- [ ] 所有间距使用 `var(--sp-*)` token，无随机数字？

**色彩与品味**：
- [ ] 只用了绿、粉、黄、褐四色？严禁五颜六色
- [ ] 粉色只用在 1-2 个最关键的点？
- [ ] 无纯黑 `#000`（用 `var(--brown)` 即 `#2D2926`）？
- [ ] 阴影染色（非灰色默认）或不用阴影？
- [ ] taste.md 合规？
